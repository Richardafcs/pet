# Definition of Ready

任务进入开发前必须满足以下条件。

## 通用条件

- 有明确需求编号或开发任务编号。
- 有用户可见结果或技术交付结果。
- 有验收标准。
- 有测试策略。
- 依赖项明确。
- 不包含未决产品问题。

## 功能任务

功能任务必须说明：

- 用户入口。
- 成功路径。
- 错误路径。
- 数据读写。
- 空状态。
- 可访问性要求。

## 算法任务

算法任务必须说明：

- 输入。
- 输出。
- 边界条件。
- 排序或评分规则。
- 可解释原因。
- 单元测试用例。

## 外部服务任务

外部服务任务必须说明：

- provider。
- API key 存放位置。
- 上传数据类型。
- 失败 fallback。
- 速率限制。
- 隐私提示。

## 不 Ready 的例子

- “做一个好看的首页”
- “接入 AI”
- “优化算法”
- “加测试”

这些任务需要拆成可验收的具体工作。
