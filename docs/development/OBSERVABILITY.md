# 可观测性

## MVP 目标

MVP 不需要复杂监控，但需要足够的调试信息来定位演示问题，同时不能泄露敏感数据。

## 日志规则

允许记录：

- 用户触发的功能名。
- mock adapter 是否返回成功。
- 计划项数量。
- 错误码。
- 性能耗时。

禁止记录：

- 小票全文。
- 原始图片内容。
- 付款信息。
- API key。
- 用户精确位置。

## 错误边界

React 应有全局 error boundary：

- 显示可恢复错误文案。
- 提供重载页面和清除本地数据入口。
- 开发模式显示更多信息。
- 生产模式隐藏内部 stack。

## 指标建议

未来可统计：

- 添加物品次数。
- 识别候选确认率。
- 计划完成率。
- 丢弃记录数量。
- 宠物状态平均值。

这些指标应匿名化，并在隐私说明中说明。

## 性能指标

本地可检查：

- 首屏可交互时间。
- 计划生成耗时。
- IndexedDB 读写耗时。
- 上传图片预处理耗时。

目标：

- 100 个库存项内计划生成小于 100ms。
- 页面主要交互不阻塞。
