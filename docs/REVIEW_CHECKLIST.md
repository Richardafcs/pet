# 文档复核清单

## 产品一致性

- [ ] README 中的一句话介绍与产品需求一致。
- [ ] MVP 范围没有超过 hackathon 可交付能力。
- [ ] 用户故事都有对应功能或后续计划。
- [ ] 非目标清楚，避免误导评委或用户。
- [ ] 食品安全文案谨慎，不承诺“安全可食用”。

## 开发一致性

- [ ] 技术设计与实际项目结构一致。
- [ ] `docs/development/PROJECT_STRUCTURE.md` 与实际目录一致。
- [ ] `docs/development/MODULE_CONTRACTS.md` 与实际导出的接口一致。
- [ ] `docs/development/REQUIREMENTS_TRACEABILITY.md` 覆盖所有 P0/P1 需求。
- [ ] `docs/development/QUALITY_GATES.md` 已转化为真实脚本或 CI 检查。
- [ ] 数据模型与代码类型一致。
- [ ] 算法文档与实现逻辑一致。
- [ ] OCR adapter 文档与真实 provider 一致。
- [ ] 环境变量和密钥说明不泄露敏感信息。
- [ ] `docs/development/TASK_BREAKDOWN.md` 中已完成任务与实际状态一致。

## 测试一致性

- [ ] 每个 P0 功能至少有一个测试用例。
- [ ] 每个 P0 功能在追踪矩阵中有模块、测试和任务。
- [ ] 计划算法有单元测试。
- [ ] 宠物状态计算有单元测试。
- [ ] 添加物品和完成计划有 E2E 测试。
- [ ] 演示路径有手动验收步骤。

## 用户体验

- [ ] 首页直接展示宠物和今日计划。
- [ ] 添加物品流程低负担。
- [ ] 识别候选项可编辑、可拒绝。
- [ ] 风险标签不只靠颜色。
- [ ] 宠物反馈温和，不羞辱用户。

## 隐私和数据

- [ ] 小票隐私风险有提示。
- [ ] 图片是否上传到外部服务写清楚。
- [ ] 用户可清除本地数据。
- [ ] 日志不保存小票全文或图片内容。
- [ ] 第三方 API 来源和限制写清楚。

## 参考资料

- [ ] 外部资料链接仍有效。
- [ ] 食品日期和储存建议引用权威来源。
- [ ] API 文档链接指向官方文档。
- [ ] 参考资料没有被写成未经验证的产品承诺。
